🔒 Parameterised queries active. The same injection payload is now passed as a literal string value — it cannot modify the SQL syntax.
🛡️ SQL Injection retrieved the row — but bcrypt blocked the login.
The injected payload modified the WHERE clause and found the user record. However, bcrypt.checkpw() was still called on the supplied password. With the wrong password, bcrypt returns False — access denied.

💡 Lesson: SQL Injection in a bcrypt app exposes data (the account row), but cannot bypass login unless the attacker also knows the real password. Try the Article Search to see UNION injection stealing hashes directly.
