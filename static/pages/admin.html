🛡️ Even with full DB access, bcrypt hashes are safe. Each hash is one-way and salted — reversing it requires brute-forcing ~212 iterations per guess.

✅ What bcrypt protects

Attacker gets the hash but cannot derive the original password. Each hash has a unique random salt embedded.

❌ What is exposed

Usernames, emails, and roles are visible. This enables phishing, credential stuffing, and social engineering.

📖 Lesson

Always use bcrypt/argon2. Never store plaintext. Even a full DB leak is survivable if hashing is done right.

-- users table
CREATE TABLE users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  username      TEXT NOT NULL UNIQUE,
  email         TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,       -- bcrypt $2b$12$...
  role          TEXT NOT NULL,
  created_at    TEXT NOT NULL
);

-- login_attempts (all attack history)
CREATE TABLE login_attempts (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  username    TEXT,
  password    TEXT,
  mode        TEXT,
  is_injection INTEGER,
  success      INTEGER,
  timestamp   TEXT DEFAULT CURRENT_TIMESTAMP
);
