📖 Key Concepts
❌ String Concatenation
User input is embedded directly into SQL. Special characters like ', --, OR change the query logic.
✅ Prepared Statements
Input is sent as data, not SQL. The ? placeholder is substituted safely — all special chars are escaped by the driver.
💡 The Fix
Always use parameterised queries. Never build SQL strings by concatenating user input.
